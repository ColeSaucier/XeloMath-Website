Technical Information

XeloMath is built on Unity’s engine and Supabase cloud database. Each level is a unique scene and is loaded alongside an animation layer. The animation layer controls the loading and unloading of scenes through a gesture system: swipes up and down bring the user to different levels, swipes left reload the level, and a swipe right brings the user to a level select menu. There’s an animation that plays when no activity is measured, in case a user gets stuck—this shows navigation directions for swipes (and can be turned off). Users create an account on start with basic info and a username—no password required. This creates a user in the PostGRE database. Every time a user completes a level, their time is calculated, uploaded, and a “view” for the level’s leaderboard is maintained, sent down for the user’s leaderboard.
